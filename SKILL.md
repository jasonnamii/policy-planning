# 정책기획 풀 파이프라인

정부·지자체·선거 후보 측의 정책기획을 리서치부터 기획안 완성까지 체계적으로 수행하는 허브 스킬. 6개 프로젝트(서울·경남·부산·인천·강남구·경기도) 실전에서 검증된 워크플로우를 내장한다.

**허브+스포크 구조:** 이 파일은 라우터. 각 Phase의 상세 절차·템플릿·예시는 `references/`에 분리되어 있다. 해당 Phase 진입시 레퍼런스를 읽는다.

---

## 왜 이 스킬이 필요한가

정책기획은 "진단→논리→설계"의 3단계 사고 과정이다. 리서치 없이 기획안을 쓰면 구호성 나열이 되고, 스파인 없이 축을 전개하면 병렬 나열에 그친다. 이 스킬은 각 단계의 순서를 강제하여 "왜 이 정책이어야 하는가"가 관통하는 기획안을 만든다.

---

## 핵심 프레임워크: 축 구조 (ABCD 4축 / 3축 선택)

6개 프로젝트에서 검증된 정책 구조화 골격. 프로젝트 성격에 따라 ABCD 4축 또는 3축을 선택한다.

### ABCD 4축 (문화예술 분야 / 광역단체 전 분야 기획)

ABC는 구조적 공통, D만 도시마다 다르다.

| 축 | 역할 | 성격 |
|----|------|------|
| **A** | 사람 (People) | 인재 유입·정주·성장. 전제조건 — 사람 없으면 나머지 공허 |
| **B** | 공간 (Place) | 인프라 재편·전환·분산. A+C+D의 물리적 기반 |
| **C** | 시장 (Market) | 글로벌 확장·수출·브랜딩. A+B 결과물의 출구 |
| **D** | 도시고유축 | 다른 축으로 흡수 불가한 고유 자산. 도시마다 완전히 다름 |

**D축 결정 체크리스트 (3문항 중 2개↑ Yes면 D축 채택):**
1. 타 도시에 부재하거나 이 도시에서만 유의미한 자산인가?
2. ABC 어느 축에도 흡수할 수 없는가? (A·B·C 각각 대입 테스트)
3. 독립 예산·조직·법적 근거로 분리 추진이 가능한가?

→ 2개↑ Yes = D축 채택. 1개 이하 = 3축으로 전환.
→ 검증 사례: 서울(시민경험), 부산(해양), 경남(광역분산), 인천(관문/공항), 경기도(분야마다 상이)

### 3축 구조 (기초자치단체 / 분야별 독립 기획)

기초자치단체(구·시·군)나 분야별 독립 기획안에서는 ABCD보다 **분야 고유 3축**이 적합하다. 기초단체는 D축(도시고유)이 불필요하거나 3축으로 충분한 경우가 많기 때문이다.

| 기준 | ABCD 4축 선택 | 3축 선택 |
|------|:---:|:---:|
| 광역단체(시·도) 전 분야 | ✅ | |
| 문화예술 단일 분야 | ✅ | |
| 기초자치단체(구·시·군) | | ✅ |
| 분야별 독립 기획안 | | ✅ |
| 다분야(6+) 통합 프로젝트 | | ✅ (분야당 3축) |

**3축 검증 사례 — 강남구 6분야:**
- 주거·도시정비: 재건축 정상화 / 2029 마스터플랜 / 주거 안전망
- 재정·자치: 공동과세 방어 / 세부담 완화 / 재정자립 2.0
- 경제·일자리: 5대 상권 살리기 / 테헤란로 AI 전환 / 2029 글로벌 비즈니스

**3축 도출 방법:** 리서치에서 도출된 3대 진단(위기/역차별/과제)에 대응하는 3개 처방 축으로 설계. 각 축은 독립 기획안 수준으로 전개.

**스킬 간 호출 방향 (순환 방지):**

**폴더 선행 생성 (Phase 0 강제):** 스킬 호출시 가장 먼저 행정단위·도시유형·분야 수를 확인한 뒤, **폴더 구조를 실제로 생성**하고 사용자에게 보여준다. 폴더가 눈에 보여야 작업 범위와 진행 상태를 추적할 수 있기 때문이다. 폴더 생성 전 리서치·기획안 착수 금지.

**사용자 확인 게이트:** Phase 0→1 (리서치프레임), Phase 1→2 (스파인) 전환시 반드시 사용자 확인.

| 축 | 검증 내용 |

## 도시유형 판별 (Phase 0 퀵 레퍼런스)

새 프로젝트 착수시 먼저 행정단위(광역 vs 기초)를 판별한 뒤, 유형을 확정한다.

| 판별 질문 | 메가시티형 | 항구2등도시형 | 광역지역형 | 관문도시형 | **기초자치단체형** | **광역메가도형** |
|-----------|----------|------------|----------|----------|:---:|:---:|
| 행정 단위 | 광역시 | 광역시 | 도 | 광역시 | **구·시·군** | **도(인구1천만+)** |
| 문화자산 수준 | 과잉 | 풍부(편중) | 절대부족 | 외형有·내용空 | **지역 편차** | **과잉+부족 공존** |
| 핵심 문제 | 시스템 부재 | 집중+단절 | 처음부터 건설 | 양극화+종속 | **광역 규제+권한 한계** | **시군 복잡성** |
| 정책 방향 | 보호+최적화 | 분산+연결+채움 | 유치+건설+연결 | 자족전환+관문활용 | **시-구 협력+자치 강화** | **세분화+균형** |

**검증 사례:** 서울=메가시티, 부산=항구2등도시, 경남=광역지역, 인천=관문도시, **강남구=기초자치단체**, **경기도=광역메가도**

## 진단 프레임 — 6유형별

| 도시유형 | 진단 프레임 | 검증 사례 |
|---------|------------|----------|
| 메가시티(서울형) | **7대 구조적 공백** | 서울시 문화예술 |
| 항구2등도시(부산형) | **3대 병목** | 부산시 문화예술 |
| 광역지역(경남형) | **3대 위기** | 경상남도 문화예술 |
| 관문도시(인천형) | **4대 역설(Paradox)** | 인천시 전 분야 |
| **기초자치단체(강남구형)** | **3대 역차별 / 3대 교착** | 강남구 6분야 |
| **광역메가도(경기도형)** | **N대 구조이슈** | 경기도 8분야 |

기초자치단체는 "역차별"(상위 단체에 의한 구조적 불이익)이, 광역메가도는 "구조이슈 N대"(남북격차·시군 세분화 등)가 적합하다.

## Gotchas

| 함정 | 증상 | 해결 |
|------|------|------|
| 리서치 스킵 | "바로 기획안 써줘"시 근거 빈약 | Phase 1 건너뛰기 금지. 최소 약식 프레임이라도 설계 |
| 스파인 미확정 | 축 나열되지만 "왜 이 묶음인가" 답 없음 | 스파인 + 사용자 확인 없이 Phase 2 금지 |
| D축 = 나머지 | D축이 ABC에 흡수되어 차별성 없음 | "이 도시에만 있는 것" 테스트. 흡수 가능하면 D축 아님 |
| 표 과잉 진단 | Section 2에 표부터 나열 → "왜" 실종 | 진단은 산문. 표는 Section 4(설계)에만 |
| 축 압축 나열 | Section 4가 축당 10줄 → 독립 기획 미달 | 축당 전수 전개 |
| 진단 프레임 혼용 | 병목/공백/역설을 구분 없이 섞음 | 도시유형에 맞는 프레임 1개 선택 후 일관 적용 |
| 예산 암산 | 합계 불일치 | 복합 산술 → Python 필수 |
| **4축 강제 적용** | 기초단체에 ABCD 적용 → D축이 억지 | 행정단위 판별 → 3축/4축 선택 분기 |
| **캠프전략 누락** | 정책만 쓰고 판세·브랜딩·위기관리 빠짐 | 선거 맥락이면 8대 기획안 체계 확인 |
| **교차검증 스킵** | 6+ 분야에서 예산·수치·용어 불일치 | Phase 2 완료 후 교차검증 5축 필수 |
| **제출청소 사후 처리** | 완성 후 청소하면 수정량 폭증 | 작성 시점부터 제출청소 기반 — 내부언어 최소화 |
| **수정4 미적용** | 기획안 버전업·제출청소에서 old 잔존(잡탕)·중복삽입·과삭제 | 레벨 판정→게이트→외과적 실행→POST_VERIFY 필수 |
