# Phase 3: 제출청소

기획안이 완성되면 외부 제출 전 13축 청소를 적용한다. 내부 작업 흔적이 남으면 전문성이 훼손되고, 수신자가 "AI가 쓴 문서"로 인식하기 때문이다.

---

## 13축 체크리스트

### 1. 헤더 메타
- 버전·날짜만 유지
- 작업자명, 내부 태그 삭제

### 2. 체인지로그
- 완전 삭제 (작업 중 이력은 외부에 불필요)

### 3. 내부 언어
- 축 번호 → 섹션명 (예: "축1" → "시민 수요 분석")
- 내부 약어 → 일반어 (예: "DNA 정합" → "후보 강점 연계")
- 코드명·작업명 → 정식 명칭

### 4. AI 흔적
- "이 섹션에서는~", "요약하면~" 등 생성형 패턴 삭제
- 불필요한 메타 코멘터리 삭제

### 5. 작업 마커
- TODO, FIXME, WIP 등 해소
- `<!-- -->` HTML 주석 삭제

### 6. 플레이스홀더
- [TBD], [FILL], [수치 확인 필요] 등 해소
- 미해소시 사용자에게 알림 (빈 채로 제출 금지)

### 7. 톤·격식
- 문서 전체 문체 통일 (정책 기획서 격식)
- 구어체·캐주얼 표현 제거
- 비문·번역체 수정

### 8. 코드 주석
- 인라인 코멘트 삭제
- 작업 지시 코멘트 삭제

### 9. 플랫폼 문법 변환
- `[!abstract]` → 볼드 텍스트
- `[!note]-` → H3 헤딩
- `[[#헤딩]]` → 표준 앵커 `[텍스트](#slug)`
- `[[파일명]]` → 필요시 삭제 또는 일반 텍스트로

### 10. 버전 흔적
- 본문 내 "v0.9에서는~", "이전 버전 참조" 등 삭제
- 초안/드래프트 마킹 삭제

### 11. 내부 경로
- 폴더 경로 삭제 (예: `_research/...`)
- Sources 링크 삭제 (내부 참조용)
- 연구 파일 링크 삭제

### 12. 포맷 정합
- 넘버링 통일 (1. / 1) / ① 혼용 금지)
- 표 스타일 통일 (border, padding, background)
- 헤딩 계층 통일 (H1→H2→H3 순차)

### 13. 숫자 정합
- 예산 합계 = 사업별 합산 검증
- 본문 수치와 표 수치 교차 확인
- 비율 합산 = 100% 검증
- 시계열 일관성 (2026→2030 순서)

---

## 개인명 처리

- 내부 기획자명 → 일반 직함 (예: "김형석 PD" → "기획팀")
- 후보명은 유지 (공개 문서)
