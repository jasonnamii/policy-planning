# 왜 이 스킬이 필요한가

정책기획은 "진단→논리→설계"의 3단계 사고 과정이다. 리서치 없이 기획안을 쓰면 구호성 나열이 되고, 스파인 없이 축을 전개하면 병렬 나열에 그친다. 

이 스킬은 각 단계의 순서를 강제하여 **"왜 이 정책이어야 하는가"가 관통하는 기획안**을 만든다.

6개 프로젝트(서울·경남·부산·인천·강남구·경기도) 실전에서 검증된 워크플로우를 내장했다.

## 절차의 순환 방지 규칙

**스킬 간 호출 방향:**
- policy-planning → research-frame: **단방향 호출만 허용**. Phase 1에서 research-frame을 호출하여 리서치를 위임한다.
- research-frame → policy-planning: **역호출 금지**. 리서치 중 기획 통찰은 `_research/기획메모_[축명].md`에 기록되어 리서치 종료 후 이 스킬로 인계된다.
- policy-planning → ceo-pipeline: **하위 호출 허용** (내부 조직운영 로드맵이 필요할 때). 역방향(ceo-pipeline → policy-planning) 금지.

## 폴더 선행 생성 (Phase 0 강제)

스킬 호출시 가장 먼저 행정단위·도시유형·분야 수를 확인한 뒤, **폴더 구조를 실제로 생성**하고 사용자에게 보여준다. 폴더가 눈에 보여야 작업 범위와 진행 상태를 추적할 수 있기 때문이다. **폴더 생성 전 리서치·기획안 착수 금지.**

## 사용자 확인 게이트

- **Phase 0→1:** 리서치프레임 확인 게이트
- **Phase 1→2:** 스파인 확인 게이트

이 게이트를 건너뛸 수 없다.
