# Phase 0: 프로젝트 셋업

새 정책기획 프로젝트 착수시 먼저 구조를 잡는다. 구조가 없으면 분야 간 일관성이 무너지고, 나중에 통합할 때 재작업이 발생하기 때문이다.

---

## 0-1. 주체 프로파일링

정책을 실행할 주체(후보·단체장·기관장)의 핵심 DNA를 추출한다.

| 항목 | 내용 | 왜 필요한가 |
|------|------|------------|
| 경력·전문성 | 직업 배경, 핵심 역량 | 정책의 신뢰성 근거. "이 사람이기에 이 정책" |
| 비전·슬로건 | 상위 비전, 핵심 전략 키워드 | 전 분야 정합성 기준선 |
| 지역 연고 | 출생지, 활동 지역, 정서적 기반 | 지역민 공감 포인트 |
| 정치력·실행력 | 의회 경험, 중앙정부 관계, 협상력 | 정책 실현 가능성 논거 |
| 차별적 관심사 | 특정 산업·분야에 대한 기존 활동 | 분야별 깊이 차등화 근거 |

**DNA 5대 패턴:** 위 5항목을 "DNA 5대"로 압축한다. 이후 모든 정책이 이 DNA와 정합하는지 검증하는 기준이 된다.

**예시 (인천 프로젝트):**

| DNA | 정책 함의 |
|-----|----------|
| 회계사 실용주의 | 숫자 기반 성과 측정, ROI 중심 사업 설계 |
| 수도권 전략가 | 서울 종속 탈피 + 수도권 허브화 양면 전략 |
| 인천 토박이 | 원도심 정서 이해, 신·구도심 균형 |
| 당대표급 정치력 | 중앙정부 협상력, 국비 확보, 제도 변경 추진력 |
| 콘텐츠산업 비전가 | K-컬처 허브, 공항경제 연계 |

## 0-2. 맥락 추출

정책이 적용될 지역/기관의 고유 맥락을 추출한다. 이 맥락이 전 분야에 관통하는 공통 제약/기회가 된다.

**추출 항목:**

| 항목 | 설명 | 예시 (인천) |
|------|------|------------|
| 지리적 특성 | 관문성, 접근성, 지형 | 인천국제공항, 연 7천만+ 이용객 |
| 역사적 자산 | 개항, 근대유산, 문화재 | 1883년 개항, 근대건축 밀집 |
| 구조적 이중성 | 신·구도심, 도농복합, 섬 | 송도·영종 vs 중구·동구·미추홀 |
| 인접 권역 관계 | 수도권, 광역 연계, 경쟁 도시 | 서울까지 1시간, GTX·KTX |
| 행정 변수 | 구역 개편, 특별법, 조직 신설 | 2026.7 영종구·검단구 신설 |
| 인구 구조 | 고령화, 청년 유출, 다문화 | 300만 도시, 젊은 인구 비중 |

## 0-3. 분야 도출·랭킹

정책 분야를 MECE로 도출하고 우선순위를 매긴다.

| 랭킹 기준 | 설명 |
|----------|------|
| DNA 정합 | 주체의 DNA 5대와 얼마나 맞는가 |
| 시민 관심 | 여론·체감도 기준 시급성 |
| 현직자 공백 | 현 체제가 못 채운 영역 (선거맥락) |
| 상호 의존 | 다른 분야의 상위 프레임 역할 여부 |

**산출물 형태:** 분야별 랭킹 테이블 (★ 평점 시각화)

## 0-4. 폴더 구조

```
프로젝트루트/
├── CLAUDE.md                    ← 총괄 컨텍스트
├── [통합가이드].md               ← 전 분야 네비게이션 (Phase 2 후반)
├── _research/                   ← 공통 리서치 (주체 DNA 등)
├── _archive/                    ← 총괄 아카이브
│
├── [분야A]/
│   ├── CLAUDE.md                ← 분야 전용 컨텍스트
│   ├── _research/               ← 분야별 리서치
│   ├── _archive/
│   └── [기획안들].md
│
└── [분야B]/ ...
```

**폴더 규칙:**
- `_archive/` 필수 — 삭제 대신 아카이빙
- `_research/` — 리서치는 기획안과 분리 저장
- 분야별 CLAUDE.md — Phase 상태·리서치프레임·분야 고유 원칙 관리

**파일 네이밍:** `[지역]_[분야]_[유형]_[하위]_v{M}.{m}.md`
- 예: `인천_문화예술_정책기획_B_공간재균형_v1.0.md`

**총괄 CLAUDE.md 필수 항목:**
- 주체 프로필 블록 (DNA 5대)
- 맥락 블록 (지역 고유 맥락)
- 분야 랭킹 테이블
- 작업 원칙 (전 분야 공통)
- 분야별 Phase 상태 체크리스트
- Changelog
