# Phase 1: 리서치

리서치는 기획의 토대다. 리서치 없이 쓴 기획안은 근거가 빈약하고, 숫자 없이 진단한 정책은 설득력이 없다.

---

## 1-1. 리서치프레임 설계

분야에 진입하면 바로 글을 쓰지 않는다. 먼저 "무엇을 어떤 관점에서 조사할 것인가"를 설계한다. 프레임이 없으면 리서치가 산만해지고, 빠진 관점을 나중에 발견하면 재작업이 필요하기 때문이다.

### 기본 6렌즈

| 렌즈 | 분석 관점 | 핵심 질문 |
|------|----------|----------|
| 수요/시민 | 시민이 체감하는 문제, 서비스 갭 | "시민이 가장 불편한 것은?" |
| 시장/경제성 | 인프라 용량, 비용 구조, 자원 제약 | "현재 자원으로 감당 가능한가?" |
| 기술/기회 | 미래 전환, 혁신 벡터, 기술 트렌드 | "3-5년 후 어떤 변화가 오는가?" |
| 규제/전략 | 거버넌스, 제도 설계, 법적 레버 | "어떤 제도가 막고/열고 있는가?" |
| 경쟁/비교 | 현직자·타 지역 성과와 공백 | "누가 뭘 했고, 뭘 안 했는가?" |
| 재정/제도 | 예산 구조, 국비, 재원 조달 | "돈은 어디서 오는가?" |

**커스터마이징:** 분야별로 렌즈를 추가하거나 교체한다.
- 문화예술: +창작자생태계, +글로벌브랜딩
- 교통: +광역연결, +물류
- 복지: +생애주기, +공공의료

### 리서치프레임 문서 구조

```
# [분야] 리서치프레임
## 축 정의 (6축)
  축1: [이름] — [렌즈] — [핵심 질문]
  축2: ...
## 렌즈 매트릭스 (축 × 렌즈)
## 데이터 소스 후보
## 가설 목록 (축당 1-2개)
```

**게이트:** → 반드시 사용자 확인 후 조사 착수

## 1-2. 축별 독립 리서치

각 축을 독립적으로 조사한다. 축 간 독립성을 유지해야 교차 분석에서 새로운 통찰이 나오기 때문이다.

### 축당 구조

| 요소 | 형식 | 내용 |
|------|------|------|
| 축 제목 | H2 | 축 이름 + 렌즈 카테고리 |
| 데이터 테이블 | 마크다운 표 | 5-10개 KPI (지표·수치·출처 3열) |
| 분석 | H3 + 산문 | 데이터 해석 2-3문단 |
| 구조적 통찰 | 볼드 문장 | 축의 핵심 진단 1문장 |

### 데이터 테이블 원칙

- 공식 출처 명시 (통계청, 연구원, 부처 자료)
- 비교 벤치마크 포함 (전국 평균, 타 지역 대비)
- 시계열 추이 포함 (최소 2개 시점)
- 정량+정성 혼합 (수치 + 주석)

**예시:**

```markdown
| 지표 | 수치 | 출처 |
|------|------|------|
| 문화예산 비율 | 1.8% (전국 평균 2.3%) | 2025 지방재정연감 |
| 창작공간 수 | 12개소 (서울 87개소) | 문체부 실태조사 2024 |
```

## 1-3. 스파인 도출

축별 리서치가 완료되면 축을 횡단하여 핵심 통찰을 추출한다. 스파인이 없으면 기획안이 축 나열에 그치고, "왜 이 정책 묶음인가"라는 질문에 답하지 못하기 때문이다.

### 도출 3단계

**① 핵심 통찰 3가지**
- 6축에서 반복 등장하는 패턴, 모순, 기회를 3개로 압축
- 각 통찰: **볼드 1문장** + 설명 1줄
- 흐름: 진단 → 방향 전환 → 레버/기회

**예시:**
1. **"현금에서 그물로"** — 현금 복지에서 직접 돌봄 인프라로 전환
2. **"사후 치료에서 사전 예방으로"** — 사후 대응 의료 투자에서 예방 중심으로
3. **"백지 위에 모범을 그리다"** — 행정구역 개편을 기회로 전환

**② 스파인 후보 2-3개**

| 스파인명 | 서사 논리 | 축 구조 |
|---------|----------|---------|
| "돌보는 인천" | 현금→그물 전환 | A: 생애주기 / B: 공공의료 / C: 거버넌스 |
| "안전한 인천" | 예방 중심 체계 | A: 예방인프라 / B: 긴급대응 / C: 데이터 |

**③ 추천 스파인 1개**
- 선택 근거: DNA 정합성 + 시민 공감도 + 구조적 완결성
- → **반드시 사용자 확인 후** Phase 2 진입

### 리서치 문서 마감

```
*리서치 수행일: YYYY-MM-DD*
*수렴검증: 소스유형(공공통계·연구기관·언론) × 출처거리 × 접근법 = N축 충족*
*확신도: [90|70|50|30] ([근거])*
```
