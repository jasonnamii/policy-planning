# Phase 2: 기획안

스파인이 확정되면 기획안을 작성한다. 기획안은 "진단→처방→설계"의 흐름을 따른다. 표는 설계 단계에만 등장하고, 진단 단계에서는 산문으로 문제를 서술한다 — 표부터 나열하면 "왜"가 빠지기 때문이다.

---

## 기획안 8섹션 상세

### Section 1. 한 장 요약

엘리베이터 피치. 전체 기획안의 축약.

**구성:**
- 태그라인 (인용 블록): `> **「[키 메시지]」**`
- 비전 1문장
- 8행 요약 테이블: 비전 / 진단 / 전략 구조 / 핵심 논리 / 벤치마크 / 차별점 / DNA 정합

### Section 2. 진단

문제 맥락을 산문으로 서술한다. 표 나열 금지.

**2-1. 모순 제시:** 분야의 핵심 역설을 1-2문장으로 포착
- 예: "예산은 올랐지만 그물은 성기다"
- 구체적 데이터 포함 (보더 박스)

**2-2. 구조적 공백 5개:** 현 체제가 채우지 못한 빈 곳
- 각 공백: 좌측 보더 박스 (4px solid + 라이트 배경)
- 형식: **공백 N. [제목]** → 설명 문단

**2-3. 주체 적합성:** 왜 이 주체가 이 문제를 풀기에 최적인가
- 4개 플렉스 카드 (DNA 요소별)
- 각 카드: 제목(볼드·컬러) + 설명(0.9em)

### Section 3. 비전과 전략 구조

**3-1. 비전:**
- 그래디언트 배경 박스 (linear-gradient 135deg)
- 중앙 정렬, 1.4em, 흰색 텍스트

**3-2. 축별 전략 카드 (3-4개):**
- 플렉스 컨테이너 (gap: 16px, flex-wrap: wrap)
- 축당 1카드: 큰 알파벳(1.5em) + 축 이름(볼드) + 설명(0.85em)
- 컬러 코딩: 축별 고유색 (A=녹색, B=파랑, C=보라 등)

**3-3. 전략 논리 설명:**
- 회색 박스 (border: 1px solid #aaa, padding 16px)
- "A(무엇) + B(무엇) + C(어떻게) → 통합 구조" 형식

### Section 4. 축별 핵심 설계

기획안의 본체. 각 축이 독립 기획안 수준으로 전개된다. **압축 나열 금지.**

**축당 구조:**

```
### A. [축 이름]

┌ 현황→목표 수치카드
│  그래디언트 박스: 현재 상황 · 목표 상황 · 핵심 수치 (3줄, 흰색)
│
├ 문제 진단 (border 박스, 산문 0.9em)
├ 해법 논리 (border 박스, 산문 0.9em)
│
├ 사업 1~4 독립 카드
│  좌측 컬러 보더 (4px solid [축색])
│  내부: 제목(볼드·컬러) → 목표 → 전략(리스트) → 기대효과(리스트)
│
├ 벤치마크 (회색 박스)
│  국내외 3건: "도시(카테고리) — 설명, 연결점"
│
├ 축 로드맵 테이블
│  4열: 2026/2027/2028/2029-2030
│  행: 사업별
│
└ 축 KPI
```

### Section 5. 4년 실행 로드맵

전체 축을 하나의 테이블로 통합.
- 행: 연도 (2026/2027/2028/2029-2030)
- 열: 축 A · 축 B · 축 C (· 축 D)
- 셀: 해당 연도 핵심 액션 2-4개

### Section 6. 예산과 재원

| 사업 | 4년 추정(억 원) | 재원 구성 |
|------|:---:|------|
| 사업명 | 숫자 | 국비 X% + 시비 Y% + 민간 Z% |
| **합계** | **숫자** | **가중 평균** |

하단에 회색 코멘트 (0.85em): 총 규모 비교, 재원 확보 전략 1-2문장

### Section 7. 거버넌스

- 추진 조직 구조
- 중앙정부 부처별 협력 라인
- 시민 참여 메커니즘
- 2-3개 서브섹션으로 구성

### Section 8. KPI 체계

축별 마스터 KPI (1-2개) + 세부 KPI (5-8개)

## 서울시 교훈 3대 원칙 (필수 적용)

| 원칙 | 서울 문제 | 해결 |
|------|----------|------|
| 문장 품질 | 비문, 구호성 카피 반복 | 정책 기획서 문체. 언어위생 철저 |
| 스파인 우선 | 축 병렬 나열, 상위 논리 부재 | 통합 스파인 먼저 → 축은 하위 전개 |
| 맥락 흐름 | 표 나열, 섹션 간 전환 없음 | 진단→처방→설계 3단계. 표는 설계에만 |

## 옵시디언 HTML 패턴 레퍼런스

기획안 작성시 아래 패턴을 일관되게 사용한다. 옵시디언에서 올바르게 렌더링되려면 div 내부는 HTML 태그만 사용하고, 여는 태그 직후 빈 줄을 넣지 않아야 한다.

**그래디언트 박스 (비전·수치카드):**
```html
<div style="background: linear-gradient(135deg, #색1, #색2); border-radius: 12px; padding: 24px; color: #fff;">
<p style="margin: 4px 0;"><strong>현재 상황:</strong> ...</p>
</div>
```

**좌측 보더 카드 (공백·사업):**
```html
<div style="border-left: 4px solid #색; padding: 12px 16px; margin: 8px 0; background: #라이트; border-radius: 0 8px 8px 0;">
<p style="margin: 0;"><strong style="color: #색;">제목</strong></p>
<p style="font-size: 0.9em; margin: 6px 0 0 0;">설명</p>
</div>
```

**플렉스 컨테이너 (전략카드·사업카드):**
```html
<div style="display: flex; gap: 12px; flex-wrap: wrap;">
<div style="flex: 1; min-width: 220px; ...">카드</div>
</div>
```

**핵심 규칙:**
- div 내부: HTML 태그만 (`<strong>`, `<em>`, `<a>`, `<ul><li>`). 마크다운(`**`, `[[]]`) 미렌더링
- `<div>` 여는 태그 직후 빈 줄 금지 → 빈 박스 렌더링 원인
- 다크 배경 div: 텍스트 색상은 개별 태그에 직접 `style="color:#fff"` 지정
