# 교차 검증: 5축

다분야(6+) 프로젝트에서 분야 간 정합성을 검증한다. 단일 분야 프로젝트에서는 불필요하다. 분야별 기획안이 독립적으로 작성되기 때문에, 통합 시 수치·용어·정책이 충돌하는 것을 방지한다.

---

## 5축 체크리스트

### 1. 예산 정합

- 분야별 예산 합계 = 총괄 합계 확인
- Python 교차검증 필수 (암산 금지)
- 재원 구성 비율 합산 = 100% 확인
- 국비·시비·민간 비율이 분야 간 일관적인지 확인

**검증 방법:**
```python
# 분야별 예산 추출 → 합산 → 총괄 대조
total = sum(분야별_예산)
assert total == 총괄_예산, f"불일치: {total} vs {총괄_예산}"
```

### 2. 공통수치 일관

같은 수치가 여러 문서에 등장할 때 모두 일치하는지 확인한다.

**대상 수치:**
- 인구 (총인구, 세대수, 연령별)
- GRDP, 예산 총액
- 면적, 행정구역 수
- 시계열 기준연도

**검증 방법:** grep으로 수치 키워드를 전 문서에서 추출 → 불일치 탐지

### 3. 용어 통일

같은 개념에 다른 용어를 사용하면 수신자가 혼란을 느낀다.

**대상:**
- 조직명 (약칭 vs 정식명 혼용)
- 사업명 (초안명 vs 최종명)
- 정책 용어 (같은 사업을 다른 이름으로 호칭)
- 숫자 단위 (억 vs 만 vs 천 혼용)

**검증 방법:** 용어 사전을 만들고, 전 문서에서 변형 표현을 탐색

### 4. 정책 중복

분야 간 사업이 중복되거나, 역할 분담이 불명확한 경우를 점검한다.

**점검 항목:**
- 동일 사업이 2개 분야에 등장 → 주관 분야 확정
- 유사 사업 간 차이 명확한지 확인
- 예산 이중 계상 방지
- 거버넌스 중복 (같은 위원회가 여러 분야에 등장)

### 5. DNA 정합

전 분야가 후보 DNA와 일관되게 연결되는지 확인한다.

**점검 항목:**
- 각 분야 기획안의 "주체 적합성" 섹션이 DNA 5대를 참조하는지
- DNA와 무관한 사업이 포함되어 있지 않은지
- DNA 강점이 전 분야에 고르게 분배되는지 (특정 DNA만 편중 금지)

## 교차 검증 프로세스

```
PIPE: ①예산정합(Python) → ②공통수치(grep) → ③용어통일(사전대조)
     → ④정책중복(매트릭스) → ⑤DNA정합(체크리스트)
```

**순서 강제:** 예산 정합이 가장 먼저인 이유: 숫자 오류가 가장 치명적이고, Python으로 자동 검증이 가능하기 때문이다.
